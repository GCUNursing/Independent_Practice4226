# 자율실습 예약시스템 보안형 v3 기능 반영 요약

## 반영 기능

### 1. 관리자 학기 설정

관리자 화면에 `학기` 탭을 추가했습니다.

설정 항목:

- 학년도
- 학기
- 1주차 시작일
- 총 주차 수
- 운영 요일
- 시간대 기준값(기본: Asia/Seoul)

기본값은 다음 기준입니다.

- 2026학년도
- 2학기
- 1주차 시작일: 2026-08-31
- 총 16주
- 운영 요일: 월~금

학생 화면과 관리자 출석 화면은 이 설정을 기준으로 주차와 날짜를 자동 생성합니다.

---

### 2. 학생 신청 흐름 변경

학생 화면을 다음 순서로 바꿨습니다.

1. 주차 선택
2. 날짜 선택
3. 시간 선택
4. 신청

학생은 관리자 설정의 학기 주차표 안에서만 날짜를 선택합니다.

---

### 3. 시간대 공개/비공개/노출예약

관리자 `시간` 탭에서 시간대별 공개 상태를 조정할 수 있습니다.

가능한 상태:

- 비공개
- 즉시 공개
- 예약 노출

예시:

- 시간대 등록: 2026-09-10 18:00~20:00
- 노출예약: 2026-09-07 09:00
- 결과: 2026-09-07 09:00부터 학생 화면에 자동 노출

중요 보안 처리:

- 학생은 Firestore에서 `slots` 컬렉션을 직접 읽을 수 없습니다.
- 학생 화면은 `listVisibleSlots` Cloud Function으로 공개 조건을 통과한 시간대만 받습니다.
- 학생이 숨겨진 slotId를 알아내도 `applyReservation` 함수가 공개 조건을 다시 검사하여 신청을 차단합니다.

---

### 4. 학생별 자율실습 진행 통계

관리자 `통계` 탭에 학생별 집계를 추가했습니다.

표시 항목:

- 학번
- 이름
- 진행횟수
- 출석
- 지각
- 결석
- 승인 대기
- 확정
- 유효 신청
- 잔여 가능 횟수

집계 기준:

- 진행횟수 = 출석 + 지각
- 유효 신청 = 대기 + 확정
- 잔여 가능 횟수 = 8 - 유효 신청
- 반려/취소는 잔여 가능 횟수에서 제외

CSV 다운로드 기능도 추가했습니다.

## 기존 함수 중 강화된 부분

- `applyReservation`
  - 비공개 시간대 신청 차단
  - 노출예약 전 신청 차단
  - 정원 초과 방지 Transaction 유지
  - 1인 최대 8회 제한 유지

- `addSlot`
  - `visible`
  - `openAtMillis`
  필드를 함께 저장하도록 확장

## 보안 규칙 변경

학생의 `slots` 직접 읽기를 차단했습니다.

```js
match /slots/{slotId} {
  allow get, list: if isAdmin();
  allow create, update, delete: if false;
}
```

학생은 공개 시간대 목록을 Cloud Function으로만 받을 수 있습니다.
