## 검토한 핵심 흐름

- 학생 로그인 후 모바일 우선 화면 렌더링
- 학생 주차 → 날짜 → 시간 → 신청 흐름 유지
- 학생 진행횟수/유효신청/잔여가능횟수 표시
- 관리자 로그인 후 PC 대시보드형 레이아웃 렌더링
- 관리자 좌측 메뉴와 작업 영역 분리
- 관리자 모바일 접속 시 가로 메뉴로 전환되는 반응형 구조
- 기존 v3 보안 구조 유지
- 기존 Cloud Functions 호출명 유지
- 기존 Firestore Rules 구조 유지

## 실제 배포 후 반드시 해야 하는 테스트

Firebase Emulator 또는 실제 테스트 계정으로 다음을 확인해야 합니다.

1. 학생 계정으로 로그인했을 때 모바일 폭에서 주차 → 날짜 → 시간 순서가 자연스러운지
2. 학생 계정으로 PC에서 접속해도 학생용 단순 화면이 유지되는지
3. 관리자 계정으로 PC에서 접속했을 때 좌측 메뉴와 우측 작업 영역이 분리되어 보이는지
4. 관리자 계정으로 모바일 접속 시 메뉴가 상단 가로 스크롤 형태로 보이는지
5. 승인/출석/시간/학기/계정/통계 탭 전환이 정상 작동하는지
6. v3 기능인 노출예약, 공개/비공개, 학생별 통계가 그대로 작동하는지
7. 학생이 비공개 또는 노출 전 시간대를 볼 수 없는지
8. 숨겨진 slotId로 직접 신청해도 서버 함수에서 거절되는지
9. 1자리 남은 상태에서 동시 신청 시 1명만 성공하는지

## 한계
