# 자율실습 예약시스템 보안형 v4 기능 반영 요약

## v4 목적

v4는 v3의 보안형 구조와 운영 기능을 유지하면서, 사용자 유형에 맞게 화면 구조를 분리한 버전입니다.

- 학생: 모바일 중심 신청 화면
- 관리자: PC 중심 대시보드 화면
- 관리자 모바일: 승인·출석·조회 등 핵심 업무 병행 가능

## 반영 내용

### 1. 학생 화면 모바일 우선 구조

학생 로그인 후 화면을 모바일 앱 흐름처럼 재구성했습니다.

```text
주차 선택
→ 날짜 선택
→ 시간 선택
→ 신청
→ 내 신청 현황 확인
```

변경 사항:

- 학생 화면 최대 폭을 모바일 기준으로 제한
- 주차/날짜/시간 단계를 카드형 섹션으로 분리
- 신청 버튼을 모바일 터치에 맞게 크게 조정
- 내 신청 현황을 같은 흐름 안에서 확인 가능하게 구성
- 진행횟수, 유효 신청 수, 잔여 가능 횟수 표시

### 2. 관리자 화면 PC 대시보드 구조

관리자 로그인 후 화면을 PC 업무용 대시보드로 재구성했습니다.

변경 사항:

- 관리자 화면 최대 폭 확대
- 좌측 사이드 메뉴 추가
- 우측 작업 영역 분리
- 현재 탭 제목과 설명 표시
- 승인, 출석, 시간, 학기, 계정, 통계 메뉴를 관리자 콘솔 형태로 정리
- 통계, 계정, 시간관리처럼 넓은 화면이 필요한 기능을 PC에서 더 보기 쉽게 조정

### 3. 관리자 모바일 병행 구조

관리자가 모바일로 접속할 경우, PC형 좌측 메뉴가 자동으로 상단 가로 메뉴로 전환됩니다.

모바일에서 병행 가능한 기능:

- 승인/반려
- 출석 체크
- 시간대 조회
- 학기 설정 확인
- 학생 통계 조회

엑셀 업로드, 대량 계정 관리, 전체 통계 확인은 여전히 PC 사용을 권장합니다.

## 보안 로직 변경 여부

이번 v4에서는 서버 보안 로직을 변경하지 않았습니다.

유지된 구조:

- Firebase Authentication 로그인
- Cloud Functions 기반 예약 신청/취소/승인/출석/계정관리
- Firestore 직접 쓰기 차단
- 학생의 비공개/노출 전 시간대 조회 차단
- `applyReservation` 서버 함수에서 노출 조건 재검증
- 1인 최대 8회 신청 제한
- Firestore Transaction 기반 정원 경쟁 처리
